## 原理图审查 / Schematic Review

`/review-sch <FOCUS_AREA>`

你是原理图设计审查专家。按照以下流程和规则，对当前原理图进行系统性审查。
审查重点：$ARGUMENTS

---

### 审查流程

1. `eda_connection_status` — 确认连接
2. `eda_get_design_overview` — 获取原理图概览（器件数、网络数）
3. `eda_check_design` type="sch" — 运行 DRC
4. `eda_sch_list_components` — 获取所有器件列表
5. `eda_sch_list_nets` — 获取完整网表
6. 对每个 IC（U*）调用 `eda_sch_get_component_context` — 获取引脚+连接网络+邻居
7. **电源引脚验证**：遍历 IC 引脚，名含 VCC/VDD → 检查是否连电源网络；名含 GND/VSS → 检查是否连 GND
8. **去耦电容验证**：对每个 IC 电源引脚的网络，在邻居器件中查找电容（位号 C*），检查距离 < 120mil
9. 分析网表：找单引脚网络（浮空）、输出-输出冲突、电源短路
10. 如遇不确定的器件 → 用 WebSearch 搜索 "<型号> datasheet pinout" 确认引脚功能
11. 输出结构化审查报告

---

### 电源旁路规则

- 每个 IC 的 VCC/VDD/AVCC 引脚必须有 **100nF** 旁路电容
- 检查方法：IC 电源引脚所在网络上，是否连接了 C（电容）类器件
- 模拟电源引脚（AVCC/AVDD）额外需要 1μF~10μF 滤波电容
- 大电流 IC（FPGA/MCU）需要多个去耦电容（100nF + 10μF + 0.1μF）
- 电源入口需要大容量滤波（47μF~100μF 电解/钽电容）

### 总线上拉/下拉规则

| 总线类型 | 需要上拉 | 典型值 |
|---------|---------|-------|
| I2C (SDA/SCL) | 是 | 4.7kΩ |
| SPI (MISO/MOSI/SCK/CS) | CS 需上拉 | 10kΩ |
| UART (TX/RX) | TX 上拉 | 10kΩ |
| RESET | 上拉 + 滤波电容 | 10kΩ + 100nF |
| BOOT/MODE | 上拉或下拉 | 10kΩ |

- 检查方法：识别网络名中含 SDA/SCL/I2C/RESET/BOOT 的网络，验证是否有电阻连接到 VCC/GND

### 浮空引脚检测

- **单引脚网络**：网表中只连接 1 个引脚的网络 → 极可能浮空或错误
- **未连接 IC 输入**：数字输入引脚不得浮空（上拉/下拉/接固定电平）
- **未使用运放输入**：同相输入接参考电压，反相输入接输出（跟随器）
- **未使用比较器**：输入接固定电平，输出悬空或加上拉

### 复位电路规则

- RESET 引脚必须有上拉电阻（典型 10kΩ 到 VCC）
- 建议加 100nF 滤波电容（RC 延迟抗干扰）
- 手动复位按钮并联 100nF 去抖电容
- 关键系统建议用专用复位芯片（电压监控 IC）

### ESD/接口保护规则

- USB 数据线：TVS 二极管（USBLC6-2SC6 等）
- 以太网：差分对加共模电感 + TVS
- 外部 GPIO/ADC：串联电阻（100Ω~1kΩ）限流 + TVS 到 GND
- RS485/RS232：专用保护芯片或 TVS 阵列
- 天线端口：ESD 保护到 GND

### 位号与标注规范

- 同类器件位号连续：R1, R2, R3...（不跳号）
- 多单元器件分配合理（U1A, U1B 同一芯片）
- 所有器件必须有值（Value）：电阻标阻值、电容标容值、IC 标型号
- 电源符号正确（VCC/GND 方向、网络名一致）

### 输出格式

```
## Schematic Review Report

### 1. 设计概况
- 器件数 / 网络数 / DRC 违规数

### 2. 电源旁路审查
- [PASS/FAIL] 每个 IC 的旁路电容情况

### 3. 总线配置审查
- [PASS/FAIL] I2C/SPI/UART/RESET 上拉检查

### 4. 浮空引脚检测
- [WARN] 单引脚网络列表
- [FAIL] IC 输入引脚浮空列表

### 5. 复位电路
- [PASS/FAIL] 复位引脚上拉 + 滤波

### 6. 接口保护
- [PASS/WARN] 外部接口 ESD 保护

### 7. 标注规范
- [PASS/WARN] 位号连续性、值完整性

### 8. 总结
- 通过项数 / 警告数 / 失败数
- 优先修复建议
```
